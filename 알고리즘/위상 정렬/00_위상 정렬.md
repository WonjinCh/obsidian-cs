# 위상 정렬

인덱스: [[알고리즘 개요]]

---

## 개요

**DAG(방향 비순환 그래프)** 에서 간선의 방향을 거스르지 않는 순서로 정점을 나열하는 알고리즘. 선행 관계가 있는 작업 순서 결정에 사용.

---

## 핵심 아이디어

- **진입 차수(In-degree)**: 정점으로 들어오는 간선의 수
- in-degree가 0인 정점을 큐에 삽입
- 큐에서 꺼낸 정점을 결과에 추가, 인접 정점의 in-degree 감소
- in-degree가 0이 된 정점을 다시 큐에 삽입
- 큐가 빌 때까지 반복

## 동작 원리 (단계별)

```
그래프 (선수과목): 1→3, 1→4, 2→4, 3→5, 4→5
in-degree: {1:0, 2:0, 3:1, 4:2, 5:2}

Init: Queue=[1,2]

Step1: pop(1), 결과=[1]
       → 3의 in-degree: 1→0, 4의 in-degree: 2→1
       Queue=[2,3]

Step2: pop(2), 결과=[1,2]
       → 4의 in-degree: 1→0
       Queue=[3,4]

Step3: pop(3), 결과=[1,2,3]
       → 5의 in-degree: 2→1
       Queue=[4]

Step4: pop(4), 결과=[1,2,3,4]
       → 5의 in-degree: 1→0
       Queue=[5]

Step5: pop(5), 결과=[1,2,3,4,5]
```

## 예시 코드

### Java
```java
List<Integer> topologicalSort(int V, List<List<Integer>> graph) {
    int[] indegree = new int[V];
    for (int u = 0; u < V; u++)
        for (int v : graph.get(u)) indegree[v]++;

    Queue<Integer> queue = new LinkedList<>();
    for (int i = 0; i < V; i++)
        if (indegree[i] == 0) queue.offer(i);

    List<Integer> result = new ArrayList<>();
    while (!queue.isEmpty()) {
        int node = queue.poll();
        result.add(node);
        for (int next : graph.get(node))
            if (--indegree[next] == 0) queue.offer(next);
    }

    // 사이클 감지: result.size() != V이면 사이클 존재
    return result.size() == V ? result : null;
}
```

### JavaScript
```javascript
function topologicalSort(V, graph) {
    const indegree = new Array(V).fill(0);
    for (let u = 0; u < V; u++)
        for (const v of graph[u]) indegree[v]++;

    const queue = [];
    for (let i = 0; i < V; i++)
        if (indegree[i] === 0) queue.push(i);

    const result = [];
    while (queue.length > 0) {
        const node = queue.shift();
        result.push(node);
        for (const next of graph[node])
            if (--indegree[next] === 0) queue.push(next);
    }

    return result.length === V ? result : null;  // null = 사이클
}
```

## 주의사항

- **DAG에서만** 동작 — 사이클 있으면 불가능
- 위상 정렬 결과는 유일하지 않을 수 있음
