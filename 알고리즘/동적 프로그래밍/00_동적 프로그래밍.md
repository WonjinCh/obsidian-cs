# 동적 프로그래밍

인덱스: [[알고리즘 개요]]

---

## 정의

**중복되는 하위 문제**의 결과를 저장(메모이제이션/타뷸레이션)하여 재사용함으로써 효율을 높이는 알고리즘 설계 기법.

---

## 핵심 아이디어

DP가 적용되려면 두 조건을 만족해야 한다:
1. **최적 부분구조 (Optimal Substructure)**: 문제의 최적해가 하위 문제의 최적해로 구성
2. **중복 부분문제 (Overlapping Subproblems)**: 동일한 하위 문제가 반복 등장

---

## 두 가지 구현 방식

### Top-Down (메모이제이션)
```python
memo = {}
def fib(n):
    if n <= 1: return n
    if n in memo: return memo[n]
    memo[n] = fib(n-1) + fib(n-2)
    return memo[n]
```

### Bottom-Up (타뷸레이션)
```python
def fib(n):
    dp = [0] * (n + 1)
    dp[1] = 1
    for i in range(2, n + 1):
        dp[i] = dp[i-1] + dp[i-2]
    return dp[n]
```

| | Top-Down | Bottom-Up |
|--|---------|-----------|
| 구현 방식 | 재귀 + 캐시 | 반복문 + 배열 |
| 직관성 | 높음 | 낮음 |
| 스택 오버플로 | 위험 | 없음 |
| 불필요 계산 | 자동 생략 | 전부 계산 |

---

## 언제 사용하나

- 경우의 수, 최적값 등 **점화식**으로 표현 가능할 때
- 완전탐색으로 시간 초과가 발생할 때
- 하위 문제가 중복되는 구조일 때

---

## 장단점

| 장점 | 단점 |
|------|------|
| 중복 계산 제거 → 빠름 | 점화식 도출이 어려움 |
| 최적해 보장 | 메모리 사용 증가 |
| 광범위하게 적용 가능 | 상태 정의가 복잡해질 수 있음 |
