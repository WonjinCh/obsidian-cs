# 탐욕 알고리즘

인덱스: [[알고리즘 개요]]

---

## 정의

각 단계에서 **현재 가장 최선으로 보이는 선택**을 반복하여 전체 최적해를 구하는 방법.

---

## 핵심 아이디어

탐욕 알고리즘이 올바르게 동작하려면 두 조건을 만족해야 한다:
1. **탐욕적 선택 속성 (Greedy Choice Property)**: 지역 최적 선택이 전역 최적해로 이어짐
2. **최적 부분구조 (Optimal Substructure)**: 문제의 최적해가 하위 문제의 최적해를 포함

---

## 언제 사용하나

- 정렬 후 순서대로 선택하는 문제
- 최소 신장 트리(MST), 최단 경로 등 그래프 문제
- 증명이 가능한 탐욕 선택이 존재할 때

---

## 장단점

| 장점 | 단점 |
|------|------|
| 구현이 단순 | 모든 문제에 적용 불가 |
| 빠른 실행 속도 | 최적성 증명 필요 |
| 직관적 | 반례 존재 가능 |

## 비교

| | 탐욕 알고리즘 | [[00_동적 프로그래밍\|DP]] |
|--|------------|------------|
| 선택 방식 | 현재 최선만 고려 | 모든 경우 고려 |
| 속도 | 빠름 | 상대적으로 느림 |
| 최적성 | 조건부 | 항상 |

### 탐욕 vs DP 예시
- 거스름돈 문제: 동전이 배수 관계면 탐욕 가능, 임의 동전이면 DP 필요
- 배낭 문제: 분할 가능 배낭 → 탐욕, 0/1 배낭 → DP
