# 그래프

인덱스: [[알고리즘 개요]]

---

## 정의

**정점(Vertex, Node)** 과 **간선(Edge)** 으로 이루어진 자료구조. 객체들 간의 관계를 표현.

```
G = (V, E)
V = 정점 집합
E = 간선 집합 (두 정점을 연결)
```

---

## 유형 분류

### 방향성
| | 방향 그래프 (Directed) | 무방향 그래프 (Undirected) |
|--|---------------------|------------------------|
| 간선 | A → B (단방향) | A - B (양방향) |
| 차수 | in-degree / out-degree | degree |
| 예시 | 팔로우 관계, 일방통행 | 친구 관계, 양방향 도로 |

### 가중치
- **가중 그래프 (Weighted)**: 간선에 비용/거리 값 존재 → [[03_다익스트라\|다익스트라]], [[01_크루스칼\|크루스칼]], [[02_프림\|프림]]
- **비가중 그래프 (Unweighted)**: 연결 여부만 → [[01_너비 우선 탐색 (BFS)\|BFS]], [[02_깊이 우선 탐색 (DFS)\|DFS]]

### 특수 그래프
- **DAG (Directed Acyclic Graph)**: 방향 그래프 + 사이클 없음 → [[00_위상 정렬\|위상 정렬]]
- **트리**: DAG의 특수 형태 → [[트리]]
- **완전 그래프**: 모든 정점이 서로 연결 (E = V(V-1)/2)

## 표현 방법

### 인접 행렬 (Adjacency Matrix)
```
V×V 2차원 배열, matrix[i][j] = 1 (연결) / 0 (미연결)

장점: O(1) 연결 여부 확인
단점: O(V²) 공간, 희소 그래프에 비효율
```

```java
int[][] matrix = new int[V][V];
matrix[u][v] = 1;  // u→v 간선
```

### 인접 리스트 (Adjacency List)
```
각 정점마다 연결된 정점 목록 저장

장점: O(V+E) 공간, 희소 그래프에 효율적
단점: 특정 간선 존재 여부 O(degree) 확인
```

```java
List<List<Integer>> graph = new ArrayList<>();
for (int i = 0; i < V; i++) graph.add(new ArrayList<>());
graph.get(u).add(v);
```

```javascript
const graph = Array.from({length: V}, () => []);
graph[u].push(v);
```

## 주요 용어

| 용어 | 설명 |
|------|------|
| 경로 (Path) | 정점들의 연속 |
| 사이클 (Cycle) | 시작 = 끝인 경로 |
| 연결 요소 (Connected Component) | 서로 연결된 정점 집합 |
| 차수 (Degree) | 정점에 연결된 간선 수 |
| 진입 차수 (In-degree) | 방향 그래프에서 들어오는 간선 수 |
