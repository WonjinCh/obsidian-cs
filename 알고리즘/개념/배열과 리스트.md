# 배열과 리스트

인덱스: [[알고리즘 개요]]

---

## 정의

- **배열(Array)**: 동일한 타입의 원소가 **연속된 메모리 공간**에 저장되는 자료구조
- **연결 리스트(Linked List)**: 각 노드가 데이터와 다음 노드의 **포인터**를 가지는 자료구조

---

## 핵심 차이점

| | 배열 | 연결 리스트 |
|--|------|-----------|
| 메모리 | 연속적 | 비연속적 |
| 접근 | O(1) 인덱스 접근 | O(n) 순회 필요 |
| 삽입/삭제 (중간) | O(n) 이동 필요 | O(1) (위치 알 때) |
| 삽입/삭제 (끝) | O(1) amortized | O(1) (tail 포인터) |
| 크기 | 고정 (정적) | 가변 (동적) |
| 캐시 효율 | 높음 | 낮음 |

---

## 언어별 구현

### Java
```java
// 배열 (고정 크기)
int[] arr = new int[5];
arr[0] = 1;

// ArrayList (가변 크기 배열)
List<Integer> list = new ArrayList<>();
list.add(1);
list.get(0);       // O(1) 접근
list.remove(0);    // O(n) 이동

// LinkedList (이중 연결 리스트)
List<Integer> linked = new LinkedList<>();
linked.add(1);
((LinkedList<Integer>) linked).addFirst(0);  // O(1)
```

### JavaScript
```javascript
const arr = [1, 2, 3];
arr.push(4);     // O(1) amortized
arr.pop();       // O(1)
arr.shift();     // O(n) — 앞에서 제거
arr.unshift(0);  // O(n) — 앞에 추가
arr[0];          // O(1) 인덱스 접근

// 스택
const stack = [];
stack.push(1);  // push
stack.pop();    // pop

// 큐 (비효율, 대규모엔 deque 구현 필요)
const queue = [];
queue.push(1);   // enqueue
queue.shift();   // dequeue — O(n)!
```

## 자료구조 선택 가이드

```
빈번한 인덱스 접근  → 배열
빈번한 앞/중간 삽입 → 연결 리스트
스택 (LIFO)        → 배열 (push/pop)
큐 (FIFO)          → LinkedList 또는 Deque
우선순위 큐         → Min/Max Heap → [[트리]]
```
