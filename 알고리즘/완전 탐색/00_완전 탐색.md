# 완전 탐색

인덱스: [[알고리즘 개요]]

---

## 정의

가능한 **모든 경우의 수**를 탐색하여 정답을 찾는 방법. 정확성은 보장되지만 효율은 낮다.

---

## 핵심 아이디어

- 모든 후보를 빠짐없이 열거
- 각 후보가 조건을 만족하는지 검사
- 조건을 만족하는 후보 중 최적을 반환

---

## 언제 사용하나

- 데이터 크기가 충분히 작을 때 (n ≤ 10~20)
- 최적 알고리즘이 생각나지 않을 때 (검증용)
- 문제가 명확히 작은 탐색 공간을 가질 때

---

## 장단점

| 장점 | 단점 |
|------|------|
| 구현이 단순 | 시간복잡도가 매우 큼 |
| 항상 정답 보장 | 데이터가 크면 시간 초과 |
| 디버깅 쉬움 | 메모리 문제 발생 가능 |

---
