# 백트래킹

상위개념: [[00_완전 탐색]] | 인덱스: [[알고리즘 개요]]

---

## 개요

**가지치기(Pruning)** 를 통해 불필요한 탐색을 조기에 포기하는 완전탐색의 최적화 버전. 해를 찾아 나가다 조건에 맞지 않으면 이전 상태로 돌아간다.

---

## 핵심 아이디어

1. 후보를 하나씩 선택 (탐색)
2. 선택이 유망하지 않으면 즉시 포기 → **Pruning**
3. 이전 단계로 복귀 → **Backtrack**
4. 해를 찾을 때까지 반복

> 유망 함수(promising function): 현재 상태가 해로 이어질 가능성 판단

## 동작 원리 — N-Queens (4×4)

```
배치 시도: (0,0) → (1,2) → (2,?)
  → (2,0): 열 충돌 ✗
  → (2,1): 대각선 충돌 ✗
  → (2,3): (3,1) 가능 → 해 발견!
```

## 예시 코드

### Java — N-Queens
```java
int N = 8;
int[] col = new int[N];
int count = 0;

void solve(int row) {
    if (row == N) { count++; return; }
    for (int c = 0; c < N; c++) {
        col[row] = c;
        if (isValid(row)) solve(row + 1);
    }
}

boolean isValid(int row) {
    for (int r = 0; r < row; r++) {
        if (col[r] == col[row]) return false;
        if (Math.abs(col[r] - col[row]) == row - r) return false;
    }
    return true;
}
```

### JavaScript — N-Queens
```javascript
function solveNQueens(n) {
    const results = [];
    const cols = [];

    function isValid(row) {
        for (let r = 0; r < row; r++) {
            if (cols[r] === cols[row]) return false;
            if (Math.abs(cols[r] - cols[row]) === row - r) return false;
        }
        return true;
    }

    function solve(row) {
        if (row === n) { results.push([...cols]); return; }
        for (let c = 0; c < n; c++) {
            cols[row] = c;
            if (isValid(row)) solve(row + 1);
        }
    }

    solve(0);
    return results;
}
```

## 언제 사용하나

- 조합/순열 탐색에서 조건을 만족하지 않는 경로를 빠르게 제거할 수 있을 때
- N-Queens, 스도쿠, 미로 찾기 등
- 제약 충족 문제 (Constraint Satisfaction Problem)

---

## 주의사항

- 유망 함수를 빡빡하게 설계할수록 성능 향상
- 재귀 깊이가 깊으면 스택 오버플로 주의
- 방문 배열 (`visited[]`) 초기화를 빠뜨리지 말 것
