# 분할 정복

인덱스: [[알고리즘 개요]]

---

## 정의

문제를 **더 작은 하위 문제로 분할**하고, 각각을 재귀적으로 해결한 뒤 **결과를 합쳐** 원래 문제를 해결하는 방법.

---

## 핵심 아이디어

1. **Divide (분할)**: 문제를 동일한 유형의 작은 문제로 나눔
2. **Conquer (정복)**: 하위 문제를 재귀적으로 해결
3. **Combine (합병)**: 하위 문제의 해를 합쳐 원래 문제의 해 도출

---

## 언제 사용하나

- 문제가 동일한 구조의 더 작은 문제로 나뉠 때
- 하위 문제들이 **서로 독립적**일 때 (DP와의 차이)
- 재귀 분할로 효율이 향상될 때

---

## 장단점

| 장점 | 단점 |
|------|------|
| 복잡한 문제를 단순화 | 재귀 오버헤드 발생 |
| 병렬 처리에 유리 | 합병 비용이 클 수 있음 |
| 캐시 효율 좋음 | 스택 오버플로 위험 |

---

## 마스터 정리 (Master Theorem)

```
T(n) = aT(n/b) + f(n)  (a ≥ 1, b > 1)

- f(n) = O(n^(log_b a - ε)  →  T(n) = Θ(n^(log_b a))
- f(n) = Θ(n^(log_b a))    →  T(n) = Θ(n^(log_b a) · log n)
- f(n) = Ω(n^(log_b a + ε) →  T(n) = Θ(f(n))
```
