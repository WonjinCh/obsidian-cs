# UML (Unified Modeling Language)

상위개념: [[디자인패턴 개요]] | 인덱스: [[디자인패턴 개요]]

---

## 개요

소프트웨어의 구조, 행위, 상호작용을 **시각적으로 표현**하기 위한 표준 모델링 언어. 설계를 소통하고 문서화할 때 필수 도구.

---

## 주요 다이어그램 종류

| 다이어그램 | 목적 |
|---|---|
| 클래스 다이어그램 | 시스템의 정적 구조 (클래스, 관계) |
| 시퀀스 다이어그램 | 객체 간 메시지 흐름 (시간순) |
| 유즈케이스 다이어그램 | 시스템과 사용자 행위자의 상호작용 |
| 상태 다이어그램 | 객체 상태 전이 표현 |
| 컴포넌트 다이어그램 | 물리적 모듈 구조 |

---

## 클래스 다이어그램 표기법

```
▸ 클래스 구성
┌───────────────┐
│  ClassName    │  ← 클래스 이름
├───────────────┤
│ - field: Type │  ← 속성 (- private, + public, # protected)
├───────────────┤
│ + method(): T │  ← 메서드
└───────────────┘
```

---

## 관계 표기 정리

| 관계 | 표기 | 의미 |
|------|------|------|
| **상속** (Generalization) | 실선 + 빈 삼각형 화살표 | `is-a` (부모 방향) |
| **실체화** (Realization) | 점선 + 빈 삼각형 화살표 | 인터페이스 구현 |
| **연관** (Association) | 실선 | 참조 관계 (필드로 가짐) |
| **집합** (Aggregation) | 실선 + 빈 마름모 | 약한 전체-부분 (독립 생존 가능) |
| **합성** (Composition) | 실선 + 채워진 마름모 | 강한 전체-부분 (생명주기 공유) |
| **의존** (Dependency) | 점선 + 화살표 | 일시적 사용 (파라미터, 반환) |

---

## 강도 순서 (강함 → 약함)

```
합성(Composition) > 집합(Aggregation) > 연관(Association) > 의존(Dependency)
```
