# 데코레이터 패턴

상위개념: [[00_구조 패턴]] | 인덱스: [[디자인패턴 개요]]

---

## 개요

기존 객체를 훼손시키지 않은 채 내부 코루틴에 특수 래퍼(Wrapper) 껍질을 겹겹이 씌워서, 동적으로 새로운 책임/기능을 추가하는 구조 패턴. (상속의 한계를 극복하는 유연한 대안)

---

## 핵심 아이디어

1. 확장할 새 기능을 장식자(Decorator) 객체 안에 구성.
2. 컴포넌트(구체 객체)를 데코레이터가 감싸(Wrap)고 해당 데코레이터를 또 다른 장식자가 계속 감쌈 (마치 양파껍질 또는 마트료시카).
3. `super` 호출이나 하위 위임을 통해 본래 객체의 처리에 새로운 추가 처리(Decoration) 수행.

---

## 장단점

| | 특징 |
|---|---|
| 장점 | 서브클래스 상속의 거대한 트리를 만들지 않고도 객체 기능을 자유분방하게 조합, 런타임 동적 확장 수월 (OCP) |
| 단점 | 수많은 소형 클래스가 양산되고, 디버깅 시 래퍼(Wrapper) 계층이 아주 깊게 쌓여 추적이 상당히 피곤함 |

---

## 예시 코드

### Java
```java
// Component
interface Coffee { double getCost(); String getDescription(); }
class Espresso implements Coffee {
    public double getCost() { return 2.0; }
    public String getDescription() { return "에스프레소"; }
}

// Base Decorator (껍질)
abstract class CoffeeDecorator implements Coffee {
    protected Coffee decoratedCoffee;
    public CoffeeDecorator(Coffee coffee) { this.decoratedCoffee = coffee; }
    public double getCost() { return decoratedCoffee.getCost(); }
    public String getDescription() { return decoratedCoffee.getDescription(); }
}

// Concrete Decorator
class MilkDecorator extends CoffeeDecorator {
    public MilkDecorator(Coffee coffee) { super(coffee); }
    public double getCost() { return super.getCost() + 0.5; }
    public String getDescription() { return super.getDescription() + ", 우유 추가"; }
}

// 사용 (에스프레소에 우유 래핑)
Coffee coffee = new MilkDecorator(new Espresso());
```

---

## 언제 사용하나

- 객체들의 조합 수와 기능 확장의 경우의 수가 대단히 많아서, 이를 일일이 상속(서브클래스)으로 구현하면 클래스가 폭발(=Class Explosion)할 때 (예: UI 스크롤바/테두리 추가, 자바의 I/O `InputStream` 래핑구조).

---

## 주의사항

- 특정 껍질의 클래스 타입 자체에 의존적인 코드 구현을 조심해야 합니다 (데코레이터로 감싸는 순간 겉포장 타입이 돼버림).
