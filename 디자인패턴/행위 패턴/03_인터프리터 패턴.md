# 인터프리터 패턴

상위개념: [[00_행위 패턴]] | 인덱스: [[디자인패턴 개요]]

---

## 개요

어떤 언어의 **문법 규칙을 클래스 구조로 정의**하고, 주어진 문자열(문장)을 그 규칙에 맞게 해석(Interpret)하는 패턴.

---

## 핵심 아이디어

1. 문법을 `Abstract Expression` → `Terminal` / `Non-Terminal` 클래스 트리로 표현
2. 각 노드가 재귀적으로 `interpret(context)`를 호출하여 문장을 분석
3. 문법 규칙이 바뀌면 해당 클래스만 수정하거나 클래스를 추가하면 됨

---

## 예시 코드

### Java
```java
interface Expression { boolean interpret(String ctx); }

class TerminalExpression implements Expression {
    private String data;
    public TerminalExpression(String data) { this.data = data; }
    public boolean interpret(String ctx) { return ctx.contains(data); }
}

class OrExpression implements Expression {
    private Expression e1, e2;
    public OrExpression(Expression e1, Expression e2) { this.e1 = e1; this.e2 = e2; }
    public boolean interpret(String ctx) { return e1.interpret(ctx) || e2.interpret(ctx); }
}
```

---

## 언제 사용하나

- 정규 표현식, 간단한 수식 계산기, 게임 스크립트 엔진 등 소규모 DSL 구현 시

---

## 주의사항

- 문법이 복잡해질수록 클래스가 폭발적으로 늘어남 → 실무에서는 ANTLR 같은 파서 제네레이터 사용 권장
