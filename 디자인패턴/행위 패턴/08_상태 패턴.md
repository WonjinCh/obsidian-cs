# 상태 패턴

상위개념: [[00_행위 패턴]] | 인덱스: [[디자인패턴 개요]]

---

## 개요

객체의 내부 **상태(State)에 따라 행동이 바뀌는 것처럼 만드는** 패턴. 조건문(`if-else`, `switch`) 범벅 대신, 각 상태를 별도 클래스로 분리하여 처리 로직을 캡슐화합니다.

---

## 핵심 아이디어

1. 각 상태를 상태 클래스 객체(`ConcreteState`)로 표현
2. Context(원래 객체)는 현재 상태 객체에 행동을 위임 → 마치 클래스가 바뀐 것처럼 동작
3. 상태 전이(Transition)는 상태 클래스 안에서 Context 참조를 통해 결정

---

## 자료구조

- [[00_구조 패턴]] — Context가 State 인터페이스를 합성(Composition)으로 참조

---

## 예시 코드

### Java
```java
interface State { void pressPlay(MusicPlayer player); }

class ReadyState implements State {
    public void pressPlay(MusicPlayer p) {
        System.out.println("재생 시작");
        p.setState(new PlayingState());
    }
}

class PlayingState implements State {
    public void pressPlay(MusicPlayer p) {
        System.out.println("일시정지");
        p.setState(new ReadyState());
    }
}

class MusicPlayer {
    private State state = new ReadyState();
    public void setState(State s) { state = s; }
    public void pressPlay() { state.pressPlay(this); }
}
```

---

## 언제 사용하나

- 자판기, 신호등, 게임 캐릭터(걷기/뛰기/점프) 등 상태에 따라 행동이 완전히 달라지는 경우
- 상태 조건 분기(`if-else`)가 너무 많아 메서드가 비대해질 때

## 주의사항

- 상태가 5개 미만이면 단순 `enum` + `switch`가 더 간결할 수도 있음
