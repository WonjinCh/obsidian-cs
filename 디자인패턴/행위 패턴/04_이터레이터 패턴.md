# 이터레이터 패턴

상위개념: [[00_행위 패턴]] | 인덱스: [[디자인패턴 개요]]

---

## 개요

컬렉션의 내부 구조를 외부에 드러내지 않고, **그 안의 요소들을 순서대로 순회(접근)** 할 수 있게 하는 패턴.

---

## 핵심 아이디어

1. 순회 책임을 컬렉션 클래스에서 분리해 `Iterator` 객체에 위임 (SRP)
2. 클라이언트는 `hasNext()` / `next()` 인터페이스만 알면 어떤 컬렉션이든 동일하게 순회 가능
3. 필요하면 다양한 순회 방식(정순, 역순, 필터링 등)을 여러 이터레이터로 제공

---

## 예시 코드

### Java
```java
interface Iterator<T> { boolean hasNext(); T next(); }

class WaitressMenu {
    private String[] items = {"햄버거", "돈까스", "파스타"};

    public Iterator<String> createIterator() {
        return new MenuIterator(items);
    }
}

class MenuIterator implements Iterator<String> {
    private String[] items;
    private int pos = 0;
    public MenuIterator(String[] items) { this.items = items; }
    public boolean hasNext() { return pos < items.length; }
    public String next()     { return items[pos++]; }
}
```

---

## 언제 사용하나

- 배열, 연결 리스트, 트리 등 다양한 구조의 내부 표현을 숨기고 일관된 순회가 필요할 때
- 실무에서는 언어 내장 Iterator(`for-each`, `stream()`)로 대부분 해결됨
