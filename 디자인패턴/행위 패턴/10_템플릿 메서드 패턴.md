# 템플릿 메서드 패턴

상위개념: [[00_행위 패턴]] | 인덱스: [[디자인패턴 개요]]

---

## 개요

알고리즘의 **전체 뼈대(단계별 순서)를 상위 클래스에 정의**하고, 세부 구현이 달라지는 단계들만 하위 클래스에서 오버라이딩하게 하는 패턴.

---

## 핵심 아이디어

1. 공통 흐름(`final` 템플릿 메서드)을 부모에 고정
2. 변하는 부분만 `abstract` 훅 메서드로 선언 → 자식이 구현
3. 코드 중복이 부모에 모이고, 전체 흐름 제어권은 부모가 가짐

---

## 예시 코드

### Java
```java
abstract class DataProcessor {
    // 템플릿 메서드 — 흐름 고정
    public final void process() {
        readData();
        processData();
        writeData();
    }

    abstract void readData();
    abstract void processData();

    // 훅 메서드 — 오버라이딩 선택
    void writeData() { System.out.println("기본 결과 출력"); }
}

class CSVProcessor extends DataProcessor {
    void readData()    { System.out.println("CSV 읽기"); }
    void processData() { System.out.println("CSV 파싱"); }
}
```

---

## 언제 사용하나

- 알고리즘의 큰 단계는 동일하지만 세부 구현이 조금씩 다른 여러 클래스가 있을 때
- 프레임워크 라이프사이클 훅(Spring의 `AbstractApplicationContext`, Android `Activity` 생명주기 등)

---

## 주의사항

- 상속에 기반하므로 부모 클래스 변경 시 하위 클래스 전체가 영향받음
- 훅 메서드가 너무 많거나 오버라이딩 의존도가 높으면 [[09_전략 패턴]] 으로 전환 고려
