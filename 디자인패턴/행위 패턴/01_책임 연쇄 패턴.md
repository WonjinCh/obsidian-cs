# 책임 연쇄 패턴

상위개념: [[00_행위 패턴]] | 인덱스: [[디자인패턴 개요]]

---

## 개요

요청을 처리할 수 있는 처리 객체(Handler)들을 체인처럼 연결하여, **앞의 객체가 처리하지 못하면 다음 객체로 요청을 전달**하는 패턴.

---

## 핵심 아이디어

1. 각 핸들러는 공통 인터페이스를 구현하고, 내부에 `다음 핸들러(next)`의 참조를 가짐
2. 요청이 오면 자신이 처리 가능한지 확인 → 가능하면 처리 후 종료, 불가면 `next.handle()` 위임
3. 클라이언트는 체인의 첫 번째 핸들러로만 요청을 보내면 되므로 수신자를 알 필요 없음

---

## 시간복잡도 / 공간복잡도

| | 복잡도 |
|--|--------|
| 처리 | O(n) — 최악의 경우 모든 핸들러를 순회 |
| 공간 | O(1) — 체인 연결만 유지 |

---

## 예시 코드

### Java
```java
abstract class Handler {
    protected Handler next;
    public void setNext(Handler next) { this.next = next; }
    public abstract void handle(int level);
}

class InfoHandler extends Handler {
    public void handle(int level) {
        if (level == 1) System.out.println("[INFO] 처리");
        else if (next != null) next.handle(level);
    }
}

class ErrorHandler extends Handler {
    public void handle(int level) {
        if (level >= 3) System.out.println("[ERROR] 처리");
        else if (next != null) next.handle(level);
    }
}
```

---

## 언제 사용하나

- 여러 단계의 검증(인증, 권한, 캐시 확인 등)을 체인으로 구성해야 할 때
- Spring Security 필터 체인, 서블릿 필터, 로그 레벨 처리 등

---

## 주의사항

- 체인 끝까지 처리되지 않으면 요청이 그냥 버려질 수 있어, 기본(Default) 핸들러를 두거나 예외 처리를 고려해야 함
