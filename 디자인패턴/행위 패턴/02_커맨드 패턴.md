# 커맨드 패턴

상위개념: [[00_행위 패턴]] | 인덱스: [[디자인패턴 개요]]

---

## 개요

요청(행위)을 **객체 형태로 캡슐화**하여, 호출자(Invoker)와 수신자(Receiver)를 완전히 분리하는 패턴. 실행 취소(Undo), 큐잉, 로깅 기능을 추가하기 쉬워집니다.

---

## 핵심 아이디어

1. `Command` 인터페이스에 `execute()` 하나만 노출
2. 호출자(버튼, 단축키)는 `Command` 객체만 들고 있음 — 내부 Receiver(실제 로직)를 모름
3. 실행 전 상태를 저장해두면 Undo 구현이 자연스럽게 따라옴

---

## 예시 코드

### Java
```java
interface Command { void execute(); void undo(); }

class Light { public void on() { System.out.println("불 켜짐"); } public void off() { System.out.println("불 꺼짐"); } }

class LightOnCommand implements Command {
    private Light light;
    public LightOnCommand(Light light) { this.light = light; }
    public void execute() { light.on(); }
    public void undo()    { light.off(); }
}

class RemoteControl {
    private Command command;
    public void setCommand(Command c) { command = c; }
    public void pressButton() { command.execute(); }
    public void pressUndo()   { command.undo(); }
}
```

---

## 언제 사용하나

- 텍스트 에디터의 Undo / Redo 기능
- 작업을 큐에 쌓아 나중에 실행하거나, 지연 실행이 필요할 때
- GUI 버튼과 메뉴 항목에 동일한 기능을 여러 방식으로 트리거할 때
