# 미디에이터 패턴

상위개념: [[00_행위 패턴]] | 인덱스: [[디자인패턴 개요]]

---

## 개요

여러 객체가 서로 직접 통신하지 않고, **중재자(Mediator) 객체를 통해서만 메시지를 주고받도록** 하여 객체 간의 복잡한 의존 관계를 단순화하는 패턴.

---

## 핵심 아이디어

1. 컴포넌트(Colleague)들은 서로의 존재를 모르고, 오직 Mediator만 참조
2. Mediator가 이벤트를 받아 적절한 컴포넌트에게 위임 → M:N 결합을 1:N으로 축약
3. 관심사가 중재자 클래스 한 곳에 집중되어 전체 흐름 파악이 용이

---

## 예시 코드

### Java
```java
interface Mediator { void notify(Object sender, String event); }

class ChatRoom implements Mediator {
    private List<User> users = new ArrayList<>();
    public void register(User u) { users.add(u); }
    public void notify(Object sender, String msg) {
        for (User u : users) {
            if (u != sender) u.receive(msg);
        }
    }
}

class User {
    private String name;
    private Mediator mediator;
    public User(String name, Mediator mediator) { this.name = name; this.mediator = mediator; }
    public void send(String msg) { mediator.notify(this, "[" + name + "] " + msg); }
    public void receive(String msg) { System.out.println(name + " 수신: " + msg); }
}
```

---

## 언제 사용하나

- UI 폼 안의 여러 컴포넌트(버튼, 체크박스, 입력창)가 서로를 직접 참조하여 스파게티 코드가 될 때
- 채팅방, 항공 관제 시스템, 이벤트 버스 등

---

## 주의사항

- 미디에이터 클래스에 너무 많은 책임이 쏠리면 "God Object"가 될 수 있음
