# 메멘토 패턴

상위개념: [[00_행위 패턴]] | 인덱스: [[디자인패턴 개요]]

---

## 개요

캡슐화(정보 은닉)를 위반하지 않고도 **객체의 내부 상태를 스냅샷 형태로 저장**해 두었다가, 나중에 해당 시점으로 복원할 수 있게 하는 패턴.

---

## 핵심 아이디어

1. `Originator`(원복 대상) 객체가 자기 스냅샷인 `Memento` 객체를 생성하여 외부에 위탁
2. `Caretaker`(기록 관리자)가 Memento 리스트를 스택처럼 보관
3. 복원 시 Caretaker가 Memento를 꺼내 Originator에게 주면 상태 원복

---

## 예시 코드

### Java
```java
// Memento (상태 스냅샷)
class EditorMemento {
    private final String text;
    public EditorMemento(String text) { this.text = text; }
    public String getText() { return text; }
}

// Originator
class TextEditor {
    private String text = "";
    public void write(String t) { text += t; }
    public EditorMemento save() { return new EditorMemento(text); }
    public void restore(EditorMemento m) { text = m.getText(); }
    public String getText() { return text; }
}

// Caretaker
Deque<EditorMemento> history = new ArrayDeque<>();
TextEditor editor = new TextEditor();
editor.write("안녕"); history.push(editor.save());
editor.write("하세요");
editor.restore(history.pop()); // "안녕"으로 복원
```

## 언제 사용하나

- 텍스트 에디터 Undo / 게임 세이브 포인트 / 상태 스냅샷 / 트랜잭션 롤백 등

---

## 주의사항

- 스냅샷을 자주 저장하거나 대용량 상태를 보관하면 메모리 사용량이 크게 증가
