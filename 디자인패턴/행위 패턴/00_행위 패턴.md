# 행위 패턴

상위개념: [[디자인패턴 개요]] | 인덱스: [[디자인패턴 개요]]

---

## 개요

객체나 클래스 사이의 **알고리즘이나 책임 분배**에 관련된 패턴.  
특정 객체 혼자 수행하기 어려운 작업을 여러 객체로 분산하고, 그 과정에서 상호 통신(결합도)을 어떻게 낮출 지에 집중합니다.

---

## 핵심 아이디어

- **알고리즘 캡슐화**: 변경 가능성이 있는 알고리즘을 분리하여 나머지 시스템에 영향이 없도록 함
- **책임의 분산**: 제어 흐름을 나눠 각 객체가 자신의 한 가지 역할에만 집중(SRP)
- **통신 제어**: 런타임 객체 간의 복잡한 상호작용을 패턴화하여 단순화

---

## 패턴 목록

- [[01_책임 연쇄 패턴]] — 처리 객체를 체인으로 연결, 못 하면 다음으로 위임
- [[02_커맨드 패턴]] — 요청 자체를 객체로 캡슐화 (실행 취소, 큐 등 지원)
- [[03_인터프리터 패턴]] — 특정 언어의 문법 규칙을 클래스로 정의하고 해석
- [[04_이터레이터 패턴]] — 내부 구조 노출 없이 원소들을 순차 순회
- [[05_미디에이터 패턴]] — 객체 간 복잡한 통신을 중재자 객체 하나로 단순화
- [[06_메멘토 패턴]] — 캡슐화를 유지하면서 객체의 상태 저장 / 복원

- [[08_상태 패턴]] — 객체 내부 상태에 따라 행동이 달라지도록 캡슐화
- [[09_전략 패턴]] — 알고리즘을 분리 · 캡슐화하여 런타임에 교체 가능하게 함
- [[10_템플릿 메서드 패턴]] — 알고리즘 뼈대를 상위에 두고, 세부 구현은 하위에서
- [[11_비지터 패턴]] — 데이터 구조를 건드리지 않고 새로운 연산을 추가
