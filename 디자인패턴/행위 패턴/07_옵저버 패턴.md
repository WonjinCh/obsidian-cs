# 옵저버 패턴

상위개념: [[00_행위 패턴]] | 인덱스: [[디자인패턴 개요]]

---

## 개요

어떤 객체(Subject)의 상태가 변할 때, 그 객체에 등록(구독)된 **다수의 옵저버(Observer)에게 자동으로 변경 사실을 알리는** 일대다(1:N) 의존성 패턴. "발행-구독(Pub/Sub)" 모델의 기초.

---

## 핵심 아이디어

1. Subject는 Observer 리스트를 들고 있으며 `subscribe()` / `unsubscribe()` 로 관리
2. 상태 변경 시 `notifyObservers()` 호출 → 각 Observer의 `update()` 를 연쇄 호출
3. Subject와 Observer는 인터페이스로만 연결 → 느슨한 결합

---

## 예시 코드

### Java
```java
interface Observer { void update(String event); }

class EventManager {
    Map<String, List<Observer>> listeners = new HashMap<>();

    public void subscribe(String type, Observer o) {
        listeners.computeIfAbsent(type, k -> new ArrayList<>()).add(o);
    }
    public void notify(String type, String data) {
        for (Observer o : listeners.getOrDefault(type, List.of())) o.update(data);
    }
}

class EmailAlert implements Observer {
    public void update(String event) { System.out.println("메일 발송: " + event); }
}
```

---

## 언제 사용하나

- 한 객체의 상태 변경이 여러 곳에 동시에 반영되어야 할 때
- 주식 시세 알림, 뉴스레터 구독, MVC의 Model-View 동기화, 이벤트 리스너

---

## 주의사항

- 구독 해제를 빠뜨리면 메모리 릭(Leak) 발생 위험
- 옵저버가 많아지면 알림 체인의 실행 순서 예측이 어려워짐
