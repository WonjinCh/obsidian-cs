## 패턴 전체 목록

### 생성 패턴 (Creational)
*객체 생성 메커니즘을 다루며, 상황에 적합한 객체 생성 방식을 제공합니다.*
- [[01_싱글톤 패턴\|싱글톤]] — 오직 하나의 인스턴스만 보장
- [[02_팩토리 메서드 패턴\|팩토리 메서드]] — 서브클래스에서 객체 생성을 결정
- [[03_추상 팩토리 패턴\|추상 팩토리]] — 관련 객체들의 군을 생성
- [[04_빌더 패턴\|빌더]] — 복잡한 객체를 단계별로 생성
- [[05_프로토타입 패턴\|프로토타입]] — 기존 객체를 복사하여 새로운 객체 생성

### 구조 패턴 (Structural)
*클래스나 객체들을 조합하여 더 큰 구조를 만듭니다.*
- [[01_어댑터 패턴\|어댑터]] — 호환되지 않는 인터페이스를 변환
- [[02_브릿지 패턴\|브릿지]] — 구현과 추상을 분리하여 독립적으로 변형
- [[03_컴포지트 패턴\|컴포지트]] — 객체들을 트리 구조로 구성 (전체-부분)
- [[04_데코레이터 패턴\|데코레이터]] — 동적으로 객체에 새로운 책임/기능 추가
- [[05_퍼사드 패턴\|퍼사드]] — 복잡한 서브시스템을 단순한 인터페이스로 묶음
- [[06_플라이웨이트 패턴\|플라이웨이트]] — 메모리 절약을 위해 객체 공유
- [[07_프록시 패턴\|프록시]] — 다른 객체에 대한 접근 제어/대리자 역할

### 행위 패턴 (Behavioral)
*객체 간의 책임 할당과 알고리즘을 캡슐화합니다.*
- [[01_책임 연쇄 패턴\|책임 연쇄]] — 요청을 처리할 수 있는 객체를 체인으로 연결
- [[02_커맨드 패턴\|커맨드]] — 요청을 객체로 캡슐화 (실행 취소 등 구현)
- [[03_인터프리터 패턴\|인터프리터]] — 문법 규칙을 정의하고 해석
- [[04_이터레이터 패턴\|이터레이터]] — 내부 구조 노출 없이 요소 순회
- [[05_미디에이터 패턴\|미디에이터]] — 객체 간의 복잡한 통신을 중앙 집중화
- [[06_메멘토 패턴\|메멘토]] — 객체의 상태를 캡슐화 위반 없이 저장 및 복원
- [[07_옵저버 패턴\|옵저버]] — 상태 변경 시 의존하는 객체들에게 자동 알림
- [[08_상태 패턴\|상태]] — 객체 상태에 따라 행위 변경
- [[09_전략 패턴\|전략]] — 알고리즘을 분리하여 동적으로 교체 가능하게 함
- [[10_템플릿 메서드 패턴\|템플릿 메서드]] — 알고리즘의 골격을 정의하고 일부 단계를 서브클래스에서 구현
- [[11_비지터 패턴\|비지터]] — 데이터 구조 내의 원소들에 수행할 연산을 분리

## 패턴 선택 가이드

```
객체 생성 문제:
  단 하나의 인스턴스 필요     → 싱글톤
  생성 과정이 너무 복잡함     → 빌더
  동적으로 클래스를 결정해야 함 → 팩토리 메서드 / 추상 팩토리
  초기화 비용이 큰 객체 복제   → 프로토타입

구조적 문제:
  인터페이스가 안 맞음       → 어댑터
  여러 기능을 계속 덧붙여야 함  → 데코레이터
  시스템이 너무 복잡해 쓰기 힘듦 → 퍼사드
  객체가 너무 많아 메모리 부족  → 플라이웨이트
  트리 형태의 계층 구조       → 컴포지트

행위(동작) 문제:
  알고리즘을 자유롭게 교체해야 함 → 전략
  전체 흐름은 같으나 세부가 다름 → 템플릿 메서드
  객체 상태에 따라 행동이 바뀜  → 상태
  이벤트 발생 시 여러 곳에 알림 → 옵저버
  요청을 큐에 담거나 취소해야 함 → 커맨드
```
