# 팩토리 메서드 패턴

상위개념: [[00_생성 패턴]] | 인덱스: [[디자인패턴 개요]]

---

## 개요

부모 클래스에서 객체들을 생성하는 인터페이스(또는 뼈대)를 제공하고, **인스턴스를 생성하는 구체적인 책임은 서브클래스(자식)에게 위임**하는 생성 패턴.

---

## 핵심 아이디어

1. 객체 생성을 전담하는 팩토리 메서드를 추상화(인터페이스/추상클래스)함.
2. 클라이언트는 구체적인 구현 클래스를 모른 채 팩토리를 통해서 객체를 받음.
3. 새로운 객체가 필요할 때는 팩토리 클래스(Creator) 하위 서브클래스만 추가(OCP).

---

## 장단점

| | 특징 |
|---|---|
| 장점 | 객체 생성(Creator)과 사용(Product) 간의 결합도 저하, 확장성(OCP) 극대화 |
| 단점 | 새로운 유형의 제품을 추가할 때 서브클래스 상속 계층구조가 복잡해짐 |

---

## 예시 코드

### Java
```java
// Product
interface Button { void render(); }
class HTMLButton implements Button { public void render() { ... } }

// Creator (팩토리 추상 클래스)
abstract class Dialog {
    abstract Button createButton(); // 팩토리 메서드
    public void renderWindow() {
        Button btn = createButton();
        btn.render();
    }
}

// Concrete Creator
class WebDialog extends Dialog {
    @Override
    Button createButton() { return new HTMLButton(); }
}
```

---

## 언제 사용하나

- 생성해야 하는 객체의 정확한 유형이나 의존성을 미리 알 수 없을 때.
- 라이브러리/프레임워크 사용자가 내부 컴포넌트를 확장하게끔 훅(Hook)을 주고 싶을 때.

---

## 주의사항

- 제품(Product) 계층 구조가 너무 단순할 경우, 굳이 이 복잡한 패턴을 도입해서 오버 엔지니어링을 할 필요는 없음.
