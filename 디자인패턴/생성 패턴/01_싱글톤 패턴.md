# 싱글톤 패턴

상위개념: [[00_생성 패턴]] | 인덱스: [[디자인패턴 개요]]

---

## 개요

어떤 클래스가 최초 한번만 메모리를 할당(Static)하고 해당 메모리에 객체를 만들어 사용하는 디자인 패턴. 즉, **인스턴스가 오직 1개만 생성**되도록 보장합니다.

---

## 핵심 아이디어

1. 유일한 인스턴스를 저장할 정적 변수 선언
2. 외부에서 객체를 생성하지 못하도록 생성자를 `private`으로 선언
3. 전역에서 인스턴스에 접근할 수 있는 단일 정적 메서드 제공

---

## 장단점

| | 특징 |
|---|---|
| 장점 | 메모리 낭비 방지 (딱 1번 생성), 전역 상태 공유 용이 |
| 단점 | 동시성(Thread-Safe) 문제 처리 필요, SRP/DIP 원칙 위반 소지, 단위 테스트 어려움 |

---

## 동작 원리 (단계별)

1. 인스턴스 요청 발생 (게으른 초기화의 경우)
2. 내부에 저장된 객체가 있는지 확인 (`if instance == null`)
3. 없으면 새로 생성해서 저장, 있으면 기존에 저장된 객체를 그대로 반환

---

## 예시 코드

### Java
```java
public class Singleton {
    private static Singleton instance;

    private Singleton() { } // 외부 new 생성 방지

    public static Singleton getInstance() {
        if (instance == null) {
            instance = new Singleton(); // 게으른 초기화
        }
        return instance;
    }
}
```

---

## 언제 사용하나

- 시스템 전역에서 관리해야 하는 공통된 단일 자원(객체)이 있을 때.
- 인스턴스가 2개 이상 생성되면 로직에 치명적인 문제가 생길 때.
- 예: 데이터베이스 커넥션 풀, 로깅 유틸리티, 환경설정 관리자.

---

## 주의사항

- 멀티스레드 환경에서는 Thread-Safe 처리가 필수 (Double-Checked Locking, Enum, 정적 초기화 등). (안티 패턴으로 자주 분류됨)
