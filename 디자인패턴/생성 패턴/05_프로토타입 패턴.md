# 프로토타입 패턴

상위개념: [[00_생성 패턴]] | 인덱스: [[디자인패턴 개요]]

---

## 개요

인스턴스 생성의 비용(수행 시간, 자원 소모)이 너무 클 때 코드가 클래스에 결합되지 않고 **미리 생성된 기존의 인스턴스를 복제(Clone)** 하여 빠르게 객체를 생성하는 패턴.

---

## 핵심 아이디어

1. 프로토타입 인터페이스(Cloneable)를 활용하여 객체를 캡슐화
2. 생성자(`new`) 대신 복제(`clone()`)를 통해 메모리 카피 사용
3. 매우 다양한 설정값으로 복잡한 객체를 생성하는 번거로움 해소 (미리 세팅해둔 원형을 복사)

---

## 장단점

| | 특징 |
|---|---|
| 장점 | 초기 초기화 과정이 복잡한 객체 생성 시 매우 큰 성능 이점, 생성자 비용 생략 |
| 단점 | 복잡한 객체(순환 참조 객체 등)를 깊은 복사(Deep Copy) 하도록 구현하는 것은 상당히 까다로움 |

---

## 예시 코드

### Java
```java
public abstract class Shape implements Cloneable {
    private String id;
    protected String type;
    
    abstract void draw();
    
    // Prototype Clone 메서드
    public Object clone() {
        Object clone = null;
        try {
            // JVM 단체에서 제공하는 Shallow Copy
            clone = super.clone();
        } catch (CloneNotSupportedException e) {
            e.printStackTrace();
        }
        return clone;
    }
}
```

---

## 언제 사용하나

- 무거운 I/O 연산, DB 조회 등으로 가져온 커다란 데이터를 매번 새 인스턴스로 생성해야 하는 경우.
- 많은 Sub-Classing 없이 기본 객체를 미리 초기화 한 뒤 복사만 해서 성능 최적화를 요할 때.

---

## 주의사항

- 객체 내부에 동적으로 수정되는 레퍼런스(예: 리스트 컬렉션 등)가 포함되어 있다면, `Shallow Copy` 사용 시 여러 원형들이 메모리를 공유하게 되는 버그가 발생하므로 반드시 수동으로 `Deep Copy` 구현을 해야 합니다.
